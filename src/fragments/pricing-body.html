Questions?

What do BYOK and BYO LLM mean?

BYOK (Bring Your Own Key) — plug in API keys from OpenAI, Anthropic, Google, Azure, or any compatible provider. BYO LLM (Bring Your Own LLM) — run self-hosted or private models in your own environment. Bolt supports both. You control the model, the cost, and the data flow. Available on all plans. Bundled and Complete also include managed AI if you prefer zero setup.

How does Bolt connect to our tools?

Bolt uses the open Model Context Protocol (MCP) to connect to your SaaS tools — Slack, Jira, Notion, Salesforce, and more. No data migration needed.

What happens if we exceed the AI token budget?

You'll never be cut off. When you reach 80% of your monthly token budget, Bolt automatically routes to the most cost-efficient model so you stay productive. Once the budget is exhausted, additional usage is billed at $5 per 5M tokens. We'll alert you before you approach limits.

Is there a free trial?

Yes — download the Bolt desktop app and get a 7-day trial with full platform features. No credit card required, no data leaves your machine. For a production pilot, we offer a 30-day evaluation with dedicated onboarding. Credits apply toward your first annual contract.

Where does our data live?

With Absolute — entirely in your environment. With Bundled — your infra + Bolt's AI gateway (covered by DPA, zero-retention). With Complete — Bolt-managed infrastructure with your choice of region (US, EU, APAC).

Can we switch plans later?

Yes. Start with Absolute (BYOK / BYO LLM) and upgrade to Bundled or Complete anytime. Your data, integrations, and configuration carry over seamlessly.

What do we need to deploy Bolt?

Absolute and Bundled deploy via Docker Compose (single VM) or Kubernetes (HA). You’ll need a Redis instance and your identity provider configured. We provide deployment guides, Helm charts, and optional deployment setup services. Complete is fully managed — we handle everything.

How long does deployment take?

Docker Compose setups typically run in under an hour for staging. Production Kubernetes deployments with SSO, networking, and compliance review typically take 1–2 weeks with our guidance. We offer optional deployment engineering services for teams that want a turnkey handoff.

Questions?

Is Aeira Catalog really free?

Yes. Catalog is a curated company directory included with every Bolt plan — 8 core categories plus 2 custom, 50 records each. It helps employees find the right office, policy, app, or contact on day one. You only pay when you need the full search platform (Dynamic+).

What’s the difference between Catalog and Dynamic?

Catalog is a curated directory with keyword search and manual updates — think of it as your company’s quick-reference index. Dynamic is a full enterprise search platform: ingest any document or data, search with hybrid AI (keyword + semantic vector), use 20 ready-made entity templates or define your own via API, and enforce 4-level ACL per user.

Can I index any type of corporate data?

Yes. On Dynamic and above, you can ingest any document — wiki pages, runbooks, onboarding guides, knowledge base articles, internal docs, and more. The 20 built-in entity templates are a starting point for common HR/IT/ops data, not a limit. Use the API to define and ingest your own types.

What about a fully managed option?

Yes — we offer Sparcle-managed Aeira for organizations that don't want to run their own data plane. Includes SLA guarantees, dedicated isolation, and 24/7 monitoring. Contact us for pricing.

What search technology powers Aeira?

Paid tiers use hybrid search combining Elasticsearch (BM25 keyword matching) with Weaviate (semantic vector search using 768-dim embeddings). Every result is ACL-filtered per user. Catalog uses lightweight keyword search only.

How do I get data into Aeira?

All paid tiers support bulk ingestion via REST API — push CSV, JSON, or YAML documents directly. Enhanced and Federated tiers add high-throughput async ingestion via Kafka. We publish SDKs, CLI tools, and example scripts to help you build ingestion pipelines for your data sources.
