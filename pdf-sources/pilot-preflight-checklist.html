Phase C validation timeline · Acceptance criteria · Sign-off

Validation + Sign-Off.

60-day validation window starts only after Phase B sign-off. Acceptance criteria define what "successful pilot" means.
④

Phase C · Validation · Days 31 → 90

User onboarding31 → 388-20 pilot users on Bolt desktop + browser extension + PWA. Each onboarded against their assigned persona; ACL behavior validated per user.
Workflow capture38 → 60Users run real cross-source workflows. Each user logs at least 5 multi-source workflows. Sparcle observes (via shared session export) and tunes connectors.
Measurement55 → 80Measure against acceptance criteria below. Time-on-task, query latency, ACL-correctness checks, audit-chain integrity samples.
Go-live decision81 → 90Joint: review acceptance criteria. Decision: expand to production, extend pilot, or terminate. Reference-case slide drafted if expand.

Acceptance criteria — what "successful pilot" means

  1. ACL correctness. Zero recorded incidents of a user retrieving a document outside their 4-axis ACL scope. Validated by structured test set + observed user activity.
  2. Audit-chain integrity. Customer's verifier runs at week 4, 8, 12 — all three sample runs return chain-valid + signature-valid. Customer holds the chain export, not Sparcle.
  3. Latency. p95 search latency under customer-defined SLA (typical: 250-500ms on 100K-doc corpus; benchmark published before pilot end).
  4. Connector reliability. Each connector polls within its declared cadence; zero connector-side credential leaks; conformance harness green throughout.
  5. User adoption. ≥70% of pilot users active in the final 2 weeks. ≥5 multi-source workflows logged per active user.
  6. No new sub-processors introduced. Per your self-hosted Bolt commitments. Validated in compliance handbook §4.1 review.
  7. Threat-model walkthrough sign-off. CISO / SecArch confirms in writing that the deployed system matches the documented threat model + no critical residual risks remain unfunded.
